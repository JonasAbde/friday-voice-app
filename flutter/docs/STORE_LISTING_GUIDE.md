# App Store & Play Store Listing Guide

Complete guide for preparing Friday Voice Assistant for app store submission.

## Required Assets

### App Icons

**Android:**
- Adaptive icons (separate foreground + background)
- Sizes: 48dp, 72dp, 96dp, 144dp, 192dp (auto-generated by Flutter)
- Format: PNG with transparency
- Location: `android/app/src/main/res/mipmap-*/`

**iOS:**
- All sizes from 20x20 to 1024x1024
- Format: PNG, no transparency
- Location: `ios/Runner/Assets.xcassets/AppIcon.appiconset/`

**Store Icon (1024x1024):**
- Required for both stores
- PNG format
- No transparency
- No text or UI elements that will be added by system
- Should represent Friday brand clearly

### Screenshots

**Android (Google Play):**

Required sizes:
- Phone: 1080 x 2340 pixels (16:9 ratio)
- 7" Tablet: 1200 x 1920 pixels
- 10" Tablet: 1920 x 2880 pixels

Minimum: 2 screenshots, Maximum: 8 per device type

**iOS (App Store):**

Required sizes (per device):
- 6.5" (iPhone 14 Pro Max): 1284 x 2778 pixels
- 5.5" (iPhone 8 Plus): 1242 x 2208 pixels
- iPad Pro (3rd gen): 2048 x 2732 pixels

Minimum: 1 screenshot per device, Maximum: 10

### Screenshot Content Ideas

1. **Main Screen**: Voice interface with waveform
2. **Features**: Settings, integrations, calendar view
3. **Use Cases**: "Book a time", "Check schedule", "Create invoice"
4. **Danish**: Add Danish text overlays for localized version
5. **Clean UI**: Minimal text, show actual app screens

### Creating Screenshots

**Option 1: Device Screenshots**

1. Run app on device/emulator
2. Navigate to key screens
3. Take screenshots (Power + Volume Down on Android, Power + Volume Up on iPhone)
4. Transfer to computer
5. Crop/resize as needed

**Option 2: Frame Screenshots**

Tools:
- [Figma](https://figma.com) - Free design tool
- [Previewed](https://previewed.app) - Device mockups
- [Screenshot.rocks](https://screenshot.rocks) - Free mockup generator

**Option 3: Flutter Screenshot Tool**

```bash
flutter pub add screenshots
# Configure in screenshots.yaml
flutter pub run screenshots
```

### Feature Graphic (Android Only)

- Size: 1024 x 500 pixels
- Format: PNG or JPEG
- No transparency
- Used as hero image in Play Store
- Should showcase Friday branding and main feature

### Promotional Video (Optional but Recommended)

- Length: 15-30 seconds
- Show: Voice interaction, key features, value proposition
- Format: MP4 or MOV
- Aspect ratio: 16:9 or 9:16 (vertical for mobile)
- Tools: iMovie, DaVinci Resolve (free), Adobe Premiere

## Privacy Policy

**Required for both stores!**

Must include:
- What data you collect (voice recordings, calendar data, contact info)
- How you use it (AI processing, scheduling, invoicing)
- Data storage (where, how long)
- User rights (access, deletion, portability)
- Third-party services (OpenAI, Google Calendar)
- Contact information

Template: See `PRIVACY_POLICY_TEMPLATE.md`

Host on: Your website, GitHub Pages, or use [app-privacy-policy-generator.com](https://app-privacy-policy-generator.com)

## App Store Connect / Play Console Setup

### Google Play Console

1. **Create app**: Select default language (Danish or English)
2. **App details**:
   - App name: Friday Voice Assistant
   - Short description
   - Full description
   - App icon
   - Feature graphic
   - Screenshots
3. **Categorization**:
   - Category: Productivity
   - Tags: Voice, Assistant, Business
4. **Contact details**:
   - Email: support@rendetalje.dk
   - Website: Your website
   - Privacy policy URL
5. **Store presence**:
   - Target age rating: 3+ (everyone)
   - Content rating questionnaire (no violence, ads, etc.)
6. **App content**:
   - Privacy policy
   - Ads: No (if no ads)
   - Target audience
   - Data safety section
7. **Pricing**: Free or Paid
8. **Countries**: Denmark, or worldwide

### Apple App Store Connect

1. **Create app**: My Apps → + → New App
2. **App Information**:
   - Name: Friday Voice Assistant
   - Subtitle
   - Category: Productivity / Business
   - Content Rights: Yes (if you own it)
3. **Pricing and Availability**:
   - Price: Free or select tier
   - Availability: All countries or select
4. **Version Information** (1.0):
   - Screenshots (all required sizes)
   - Description
   - Keywords
   - Support URL
   - Marketing URL (optional)
   - Promotional text
5. **App Privacy**:
   - Data types collected
   - Data usage
   - Data linking (if tracking across apps)
6. **App Review Information**:
   - Contact: Your email/phone
   - Demo account (if login required)
   - Notes: Explain app functionality
7. **Version Release**:
   - Automatic or Manual release after approval

## Content Ratings

### Google Play (IARC):

Fill out questionnaire honestly:
- Violence: None
- Sexual content: None
- Language: None
- Controlled substances: None
- Gambling: None
- User interaction: Yes (if users can communicate)
- Data sharing: Yes (if sharing to third parties)

Result: Likely "Everyone" or "3+"

### Apple App Store:

- Age rating: 4+
- Unrestricted Web Access: Yes (if app has browser functionality)
- Medical/Treatment Information: No
- Frequent/Intense: All "None"

## Pre-Submission Checklist

### Both Stores:
- [ ] App tested on multiple devices
- [ ] No crashes or critical bugs
- [ ] All permissions explained (microphone usage)
- [ ] Privacy policy hosted and linked
- [ ] Contact email set up and monitored
- [ ] Support URL/website active
- [ ] App icon looks good (no transparency issues)
- [ ] Screenshots show actual app functionality
- [ ] Descriptions reviewed (no typos, clear value prop)
- [ ] Correct version number and build number

### Android Specific:
- [ ] Signed with release keystore
- [ ] APK size < 50MB (or use App Bundle)
- [ ] Target SDK 34 (or latest required)
- [ ] All required permissions justified
- [ ] Data safety section completed

### iOS Specific:
- [ ] All screenshot sizes provided
- [ ] Code signed with distribution certificate
- [ ] Export compliance (no encryption = No)
- [ ] Advertising identifier: No (if not using ads)
- [ ] All required metadata in all languages

## After Submission

**Review Time:**
- Google Play: 1-3 days (sometimes faster)
- Apple App Store: 1-7 days (average 2-3 days)

**Common Rejection Reasons:**

1. **Misleading descriptions** - Describe actual functionality
2. **Missing privacy policy** - Must be accessible and complete
3. **Crashes on launch** - Test thoroughly before submitting
4. **Permissions not explained** - Justify microphone access clearly
5. **Incomplete metadata** - Fill all required fields
6. **Copyright issues** - Use only your own assets

**Responding to Rejections:**

1. Read feedback carefully
2. Fix the issue
3. Update build or metadata
4. Add notes explaining changes
5. Resubmit

**Post-Approval:**

1. Monitor reviews daily (first week)
2. Respond to user feedback
3. Fix critical bugs ASAP
4. Plan regular updates (monthly/quarterly)
5. Track analytics (installs, crashes, ratings)

## App Store Optimization (ASO)

**Keywords:**
- Include in title/subtitle (max 30 chars)
- Research: Google Trends, App Annie, Sensor Tower

**Ratings:**
- Encourage satisfied users to rate

- Fix issues mentioned in reviews

**Updates:**
- Regular updates signal active development
- Include "What's New" changelog
- Fix bugs reported in reviews

**Localization:**
- Start with Danish (home market)
- Add English for wider reach
- Consider Swedish, Norwegian (similar markets)
